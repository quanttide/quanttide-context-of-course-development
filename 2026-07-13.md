# 课程研发语境

本仓库记录 `quanttide-course` 项目研发过程中的关键结论、决策和思路演变。

## 2026-07-13

### 生产流程：单仓化 + 双格式

**结论**：`data/profile/` 作为课程内容的唯一创作源头，同时产出两种格式。

```
profile/
├── lesson1.json       ← 机器可读，对接平台服务端 API
└── lesson1/
    ├── index.md       ← 人类可读，视频脚本/文档
    ├── sense1.md
    └── ...
```

| 格式 | 消费方 | 维护频率 |
|------|--------|---------|
| Markdown | 课程开发者（写脚本、录视频） | 创作时 |
| JSON | 平台服务端（上架、分发） | 每次 MD 变更后同步 |

**原则**：内容只在 profile 里写一次，不分散到多个仓库。详见 [CONTRIBUTING](https://github.com/quanttide/quanttide-profile-of-course-development/blob/main/CONTRIBUTING.md)。

### vibe-coding 作为原型验证

**结论**：氛围编程（vibe-coding）既是第一门课，也是整套生产流程的试点项目。验证通过后，模式可复制到大数据等其他课程。

### 教程仓库结构

**结论**：教程仓库只含 `docs/`（文字教程/视频脚本）和 `examples/`（示例代码），视频成品不入库、单独发布。

```
tutorials/vibe-coding/
├── docs/         ← 开源文字教程，也是视频录制脚本
├── examples/     ← 配套示例代码/资源
└── README.md
```

### Scene 数据模型扩展

**结论**：Scene 增加 Title、Steps（有序操作步骤）、VerifyTip（验证方式），让前端在视频下方同步展示操作指引。

```json
{
  "id": "vibe-s4",
  "title": "Zed Agent 配置 DeepSeek 密钥",
  "videoUrl": "vibe-coding/lesson1/configure-zed.mp4",
  "steps": [
    { "order": 1, "content": "点击星星图标打开 Zed Agent" },
    { "order": 2, "content": "选择模型 → Configure → LLM Provider" }
  ],
  "verifyTip": "输入 hello，正常回复即配置成功",
  "choices": []
}
```

**待办（未实现）**：Step 未来可支持 `errorSteps` 异常分支，在步骤内嵌故障排查树，无需跳转到独立场景。

### API 路由重组：资源嵌套

**结论**：Scenes 嵌套到 Lessons，Phases 嵌套到 Courses，URL 直接表达资源层级关系。

```
# 旧                             # 新
/scenes?lessonId={id}            /lessons/{id}/scenes
/scenes/{id}                     /lessons/{id}/scenes/{sceneId}
/phases?courseId={id}            /courses/{id}/phases
/phases/{id}                     /courses/{id}/phases/{phaseId}
```

- 路径即体现归属，不再依赖查询参数
- POST 自动从路径推断父级 ID，请求体更简洁
- 所有操作都带有父级上下文，不存在游离的资源

### 视频测试分层

**结论**：用三层测试覆盖视频服务：

| 层 | 验证内容 | 工具 |
|----|---------|------|
| HTTP | Content-Type / Range/206 / 404 | Python urllib |
| 解码 | 视频格式 / 编码器 / 分辨率 / 时长 | ffprobe |
| 浏览器 | 真实解码渲染 + oncanplay 事件 | Playwright（已移除，待需要时重新引入） |

> Playwright 浏览器测试在初期被移除，待平台前端开发阶段再重新引入。

### 愿景：从视频播放器到交互式操作指南平台

**结论**：v0.0.3 的核心方向是让课程平台从被动的"视频播放器"升级为主动的"可交互操作指南平台"。

#### 学员端形态

```
视频播放区         步骤面板
┌──────────────┐  ┌──────────────────┐
│              │  │ □ Scene 1 安装   │
│  正在播放     │  │  ① 下载 Zed      │
│  第3步/共6步 │  │  ② 安装           │
│              │  │ ➜ ③ 初始配置 ←当前 │
│              │  │     ⚠ 找不到图标？ │
│              │  │     (异常展开)     │
│              │  │ □ Scene 2 注册    │
└──────────────┘  └──────────────────┘
```

- 视频自动跳到当前步骤对应的时间位置（时间锚点）
- 右侧步骤面板高亮当前步骤
- 遇到问题时展开异常分支就地解决，无需跳转场景

#### API 即内容导航

路由重组 + name slug 查询使 URL 成为人类可读的导航路径：

```
/lessons/develop-environment-setup/scenes/zed-install
```

不仅 RESTful，还可直接作为教程文档的永久分享链接。

#### 生产链路

```
profile（写 MD 脚本）
  → lesson.json（含 Steps / errorSteps / 时间锚点）
  → 平台 API 导入
  → 学员端自动渲染为带步骤面板的交互式视频课
```

vibe-coding 先试点跑通，后续复制到其他课程。
