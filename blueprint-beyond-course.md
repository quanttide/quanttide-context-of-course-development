# 课程蓝图生成器的意外发现

## 发现

`qtcloud-course` CLI 的 `course` 命令本为生成课程蓝图设计，但实际验证中发现：
**凡是具有教学性质的场景，都可以用同一套方式从真实生产中提取结构化内容。**

关键不在于输出格式是"课程结构"还是"手册结构"，而在于内容的目标是**教会别人某件事**。只要是教学性质的，底层流程都一样：采集生产素材 → LLM 提炼 → 结构化输出。

## 边界

**适合**：需要结构化的教学内容。课程、指南、手册、教程——有清晰的层级关系，可以按 Program → Course → Phase → Lesson → Scene 这类树状结构组织。

**不适合**：需要叙事和故事线的创意写作。案例故事、随笔、复盘叙事——这些依赖情节递进和情感张力，强制套结构化模板反而损害表达。

## 可覆盖的教学场景

| 场景 | 输入素材 | 输出用途 |
|------|----------|----------|
| 正式课程 | 源码、架构文档、context/ | 教学大纲 |
| 新人 onboarding | 项目 README、CONTRIBUTING、CHANGELOG | 上手指南 |
| 最佳实践教程 | 工程实践文档、历史决策记录 | 知识沉淀 |
| 培训手册 | 操作流程、SOP | 培训材料 |
| 技术分享 | 架构设计、复盘报告 | 分享提纲 |

## 含义

`course` 这个命名窄化了 CLI 的能力。它不仅是一个课程生成器，更是一个**从真实生产中提取教学内容的工具**。只要目标是"教会人"，就能套用。

## 决定

保持现状，不泛化。先用课程场景打磨好底层流程。
