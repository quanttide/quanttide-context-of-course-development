# 课程蓝图生成器的意外发现

## 发现

`qtcloud-course` CLI 的 `course` 命令本为生成课程蓝图设计，但实际验证中发现：
给定不同的提示词和原始资料，同一套结构化输出骨架可以生成远不止课程的内容。

## 能力边界

底层是 LLM + 结构化 JSON 输出，不限于"课程"。

| 输入 | 输出格式 | 可产出 |
|------|----------|--------|
| 源码+架构文档 | 五层课程结构 | 教学大纲 |
| 公司制度+流程文档 | 章节式手册 | 员工手册 |
| 产品功能文档+API | 功能分类结构 | 用户手册 |
| CHANGELOG+Release记录 | 时间线/分类结构 | 版本发布说明 |

## 含义

`course` 这个命名反而窄化了 CLI 的能力。它本质是一个**结构化内容生成器**，课程只是应用的第一个场景。

## 决定

保持现状，不泛化。先用课程场景打磨好底层流程。
