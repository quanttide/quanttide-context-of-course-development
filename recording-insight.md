# 录播即精加工直播

## 核心思路

录播课就是处理过的直播录像，没有本质区别。产品管线是：

```
现在：第三方录制（腾讯会议/Zoom）→ 导入 Studio → 关联课时 → 播放
未来：直播自动录制 → 存入 Studio → 关联课时 → 播放
↓
在时间轴上标注 Scene/Step → 沉淀为课程资产
```

底层干的是一样的活，只是输入源不同（文件 vs 实时流）。

## 对模型的推论

**不需要新增"录播资产"模型**。Lesson 里一个 `videoUrl` 就够了——给它加个来源标记字段区分"导入文件"和"未来直播录制"即可。

## 当前阶段要做的事

1. **播放器从占位文字升级为真实播放器**——当前 PreviewScreen 的 videoUrl 只展示文字，需要改为 video_player 或等效方案
2. **"关联录制文件"操作**——文件选择 + 路径记录到 videoUrl，复用现有导入模式
3. **Scene 标注**——当前硬编码在 JSON 中，以后改为在播放器时间轴上打点

三件事都不依赖新模型，也不依赖直播基础设施。v0.0.5 或 v0.0.6 均可容纳。

## 与现有产品的关系

- 课程树 CRUD → 定义内容结构，录播素材填入 Lesson
- PreviewScreen → 从占位变为真实视频播放 + Scene 标注入口
- 导入导出 → 新增"导入录制文件"操作
- Assessment → 不受影响，考核仍独立
